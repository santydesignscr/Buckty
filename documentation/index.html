Getting Started

Welcome To Buckty

Easy to use , easy to upload , easy to launch

Welcome to Buckty documentation. I will explain you how to get buckty up and running with-in few minutes. Before proceeding to the installation please double check if you have the following modules installed.

  • PHP version 5.2.4 or newer.
  • Apache mode rewrite | enabled
  • MySQL (5.1+) via the mysql (deprecated), mysqli and pdo drivers

Note:
Please make sure you increase your server's following limits and change them to high as possible.

  • Memory limit : click here to check how to do it
  • Increase post_max_size & upload_max_filesize limits: click here to check how
  • Change max_execution_time to 0 click here to check how

Api introductions (Important):

Below is the api list used inside buckty's base system.

  • Google plus api & Google drive api
  • Dropbox api
  • Twitter api
  • Facebook api
  • Pexels api

Api Notice: It's important that you read this part of documentation. Dropbox api requires your site to use ssl protocols (https), else you can't use their api and it will result into an error or you can disable it from backend to avoid errors. We highly recommend you to use HTTPS as it secures the credentails of users and you have alot of other advantages. if you want to use free ssl cloud service we recommend you to use cloudflare.com Cloudflare provides you free to use secure ssl service and also cdn hosting network But it's file upload is limited to 100mb. if you don't want to use https then you have to Disable dropbox api, as it will not work without having https. You can use Google drive api without https (but still https is recommended).

Installation

Connect to your server and upload all the files from folder Buckty pack to your server 's root folder or any sub folder where you want to install the buckty.

Now if you visit your site link where you have uploaded buckty , you will see Installation directory is present, so it says all , you have to go to the installation path to install buckty , so do it by going to :
http://yoursite.com/install
Now you will see a bit of information before proceeding to installation.

Click on Proceed button
Now you will have to fill up all the information for your site. All of the above information is required information , so please fill it carefully. After installation you can change it inside admin dashboard.

The introdcution of fields:

Site url :


Url of the page where you are going to host your app , make sure it ends with slash at end "/"

Site path (important):

If your site is located inside a subfolder you have to enter sub folder name inside this field. Please take a look at the situations below:
If you are pointing an subdomain to your subfolder -
    for sub.domain.com - let the field be empty, do not add anything into the site path field then

    for domain.com/sub - in this case please enter the sub folder name into site path field.

Site name:

This should be a title for your site.

Site admin email address:

This should be an email for outgoing emails , so basically it can be : info@yoursite.com or no-reply@yoursite.com

Site keywords:

Keywords for meta tags , it's good for your site if you fill these keywords devided by comma (,).

Site description:

Description for your site meta description tag , to be shown on search engines, under your site link.

Site home tagline , Big Heading:

It's big heading on homepage under the logo, choose the short and described one.

Site home description , Big Heading:

It's description on homepage under the big heading, choose the short and described one.

Site upload limit (in bytes):

This should be a total space for every user to be used for uploading her/his files. You can convert the quotes you want to bytes from the following site: Converter, You should be using Bytes for the field. (it is must that you use bytes).


Site upload limit (in bytes):

This should be a max file size to be uploaded in uploading process. You can convert the quotes you want to bytes from the following site: Converter, You should be using Bytes for the field. (it is must that you use bytes).
Important:
Please enter the values according to what post_max_size & upload_max_file_size in php.ini limits , else if you want to use the higher upload limits you have to change the:
post_max_size: 8mb;
upload_max_file_size: 8mb;
You have to change the above values inside php.ini to higher numbers , else if you don't change inside php.ini and inside buckty's settings you state some higher number , it can result into an error.

Allowed file extensions:
The allowed extensions for files to be uploaded , if you want to allow specific files they must need to be devided by comma (,) eg: jpeg,png,zip,doc,pdf, else if you want to allow all types you can use *.

Disqus Comments (shortname):
This field should contains disqus commenting system 's shortname , you can get your site shortname by registering to the following link : Disqus site registration

Mail settings:
We highly recommend using : smtp2go.com as this services was tested and used with buckty, but you can use any other service you want to use for your mails. It is required that you use the smtp service , this will be used to:
  • Send registration emails
  • Send recovery emails
  • Send sharing information emails and others also.
SMTP HOST (eg: ssl://mail.smtp2go.com/):
In this field you should provide a host name of smtp service , it is highly recommended that you use secure hostname as this example: ssl://mail.smtp2go.com/ , as it will work without any problem.

SMTP port (eg: 465):
In this field you should fill the smtp port, every website can have different ports and for ssl smtp port you can use one of the following ports: 25, 465, 587

SMTP user (eg: no-reply@buckty.com):
In this field you should fill the username for smtp service.

SMTP user (your password):
In this field you should fill the password for smtp service.


Now you are good to proceed with clicking Validate button you will be taken to database host information form. Database information:
In this section you should provide valid database details for the buckty to be installed on your server, make sure you have database created or a permission for creating one , as if you have not created a database yet buckty will autoamtically create one with name you provided in form.


Now the next form you will see is admin credentials, Make sure you use secure password, the account will be created for both side admin and normal dashboard so you can use both sides at same time.

Now at the end of installation you have to rename your install folder inside root of the buckty, else the site will not work , until you not remove or rename the folder.

Buckty api introduction

Buckty supports little api that can allow users to get their own information and informations about their files into json array, here i will show you how any user can access their files and folders information and also login / update via api.
Below are the example parameters buckty require to give an access to api. 

b_auth -> Api authorization key
 
b_user -> Api user key

https://yoursite.com/api/files?b_auth=DoiH8kp0znRK6pFmOH2ZNifB4vP40s&b_user=NifB4vP40s

Users can get their api details by going to : Settings And click on Api Access button inside their settings page.
if the user haven't yet generated an api key , he will see a generate button to generate the api key , user have to click on Generate key button to generate the api key, and use that api to access api.
Upload file via api:
Users can upload the file with api by using their api key and user key. Take a look below about how it can be done:
Parameters required to upload:

  • b_auth - Authrization key
  • b_user - User key
  • absoluteUrl - link to the file (if not the file wil not be uploaded

Url to upload file:
https://yoursite.com/api/request? b_auth=DoiH8kp0znRK6pFmOH2ZNifB4vP40s &b_user=NifB4vP40s &absoluteUrl=(link)

list files / folders:
Parameters required to get list:
  • b_auth - Authrization key
  • b_user - User key

For file:
                 
https://yoursite.com/api/files? b_auth=DoiH8kp0znRK6pFmOH2ZNifB4vP40s &b_user=NifB4vP40s


For folders:

https://yoursite.com/api/folders? b_auth=DoiH8kp0znRK6pFmOH2ZNifB4vP40s &b_user=NifB4vP40s

User details:
Parameters required to get list:
  • b_auth - Authrization key
  • b_user - User key

               
https://yoursite.com/api/user? b_auth=DoiH8kp0znRK6pFmOH2ZNifB4vP40s &b_user=NifB4vP40s
